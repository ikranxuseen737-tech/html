Web Development Terminology

Core Web Development Terminology

HTML (HyperText Markup Language)
The standard markup language used to create the structure and backbone of web pages.
CSS (Cascading Style Sheets)
A style sheet language used for describing the presentation, design, and layout of a document written in HTML.
JavaScript
A high-level programming language that enables interactive and dynamic behavior on websites.
DOM (Document Object Model)
A programming interface for web documents that represents the page so that programs can change the document structure, style, and content.
HTTP (HyperText Transfer Protocol)
The foundation of data communication for the World Wide Web, defining how messages are formatted and transmitted.
API (Application Programming Interface)
A set of rules and protocols that allows different software applications to communicate and share data with each other.
Frontend
The "client-side" of a website, referring to everything users see and interact with directly in their browser.
Backend
The "server-side" of a website, consisting of the server, application, and database that manage data and logic behind the scenes.
Full Stack
A term referring to a developer who is proficient in both frontend and backend development.
Database
An organized collection of structured information, or data, typically stored electronically in a computer system.
Responsive Web Design
An approach to web design that makes web pages render well on a variety of devices and window or screen sizes.
Git
A distributed version control system used to track changes in source code during software development.
Repository (Repo)
A central location (often on platforms like GitHub) where all the files and the history of a project are stored.
Framework
A pre-written collection of code that provides a foundation for developers to build applications more efficiently (e.g., React, Vue, or Django).
Server
A computer program or device that provides functionality or data to other programs or devices, called "clients."
URL (Uniform Resource Locator)
The specific web address used to locate a resource on the internet.
Deployment
The process of pushing your code from a local environment to a live server where it can be accessed by users.
DNS (Domain Name System)
The internet's system for converting human-friendly domain names (like google.com) into numeric IP addresses.
Semantic HTML
The use of HTML markup that reinforces the meaning of the information in web pages rather than just its appearance (e.g., using <header> instead of <div>).
Debugging
The systematic process of finding and resolving bugs or defects within a program's code.
